# Code of Conduct

## 我们的承诺

为了让这个项目对所有贡献者都保持开放、专业和可协作的氛围，项目维护者承诺：

- 尊重不同背景、经验和观点
- 以建设性方式讨论产品、代码和设计方案
- 对问题聚焦在事实、复现和改进路径，而不是针对个人
- 在出现分歧时优先追求清晰、可验证和可执行的结论

## 期望的行为

参与本项目时，鼓励以下行为：

- 使用清晰、具体、专业的表达
- 在提 Issue 或 PR 时尽量提供上下文、复现步骤和验证结果
- 接受合理的技术质疑，并愿意补充依据或修正方案
- 尊重维护者对范围、优先级和架构一致性的判断
- 在公开讨论中保护他人隐私、凭据和敏感数据

## 不可接受的行为

以下行为不被接受：

- 人身攻击、羞辱、骚扰、歧视性言论
- 恶意刷屏、挑衅、引战或故意制造对立
- 未经允许公开他人的隐私信息、账号信息、服务器信息或敏感数据
- 在明知存在安全风险时公开发布利用细节、密钥或攻击脚本
- 与项目目标无关的重复争论，或阻碍正常协作的行为

## 适用范围

本公约适用于：

- GitHub Issues
- Pull Requests
- Discussions
- 文档、截图、演示材料
- 与本项目相关的其他公开协作场景

## 处理方式

项目维护者有权对不符合本公约的内容进行编辑、隐藏、锁定或删除，并视情况拒绝相关贡献。

对轻度问题，通常会先提醒并要求修正；对重复性或严重违规行为，维护者可以直接限制互动或拒绝继续协作。

## 报告渠道

如果你发现某些互动违反本公约，建议通过私下渠道联系项目维护者，或在必要时通过 GitHub 平台提供的举报 / 管理机制处理。

如果问题同时涉及安全风险，请优先参考 [SECURITY.md](./SECURITY.md)。

## 说明

本文件参考开源社区常见的协作治理实践，并结合当前项目的自托管、隐私和安全要求进行了简化。
